## Featured Project

### [STR Revenue Intelligence — OTA Performance Analysis](https://github.com/agrisigabriel/str-revenue-intelligence)

Full-cycle analysis of OTA performance for a mid-size STR management company (123 properties, $3.1M revenue, 1,507 reservations).

**The question:** How does Direct Booking compare to Airbnb, VRBO, and Booking.com — and how can the business grow its most profitable channel?

**Key findings:**
- Direct Booking generates **$697/night** vs $560 for Airbnb (24% higher)
- Despite only 23% of reservations, Direct Booking captures **28.5% of revenue**
- Direct Booking share peaks at **33% in High season** but drops to 21% in Low season

**Pipeline:** Raw Excel Data → Python (pandas) → SQLite → SQL Analysis → Power BI Dashboard

**Tools:** Python | SQL | SQLite | Power BI

## In Development

### Revenue Department — AI-Native Operating Model

Designing an AI-first revenue operations framework for short-term rental companies. The concept: **3 humans + 6 AI agents** managing 250+ units with 90% autonomous decision-making.

| Agent | Function |
|---|---|
| Dynamic Pricing Agent | Real-time rate adjustments across all units within strategy guardrails |
| Forecasting Agent | Rolling 90-day demand forecasts per property and market cluster |
| Distribution Optimization Agent | Channel mix optimization based on net revenue, not volume |
| Onboarding Agent | Automated property-to-live-listing pipeline (48h vs industry avg 7-14 days) |
| Performance Monitoring Agent | Daily unit-level diagnostics against cluster benchmarks |
| Competitive Intelligence Agent | Real-time comp set tracking and market positioning |

The model scales from 250 to 1,000+ units by adding properties, not people. AI absorbs work that scales with unit count; humans focus on strategy, relationships, and exceptions.

*Status: Architecture & framework design phase*
